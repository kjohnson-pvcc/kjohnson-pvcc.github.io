    • Software Life Cycle: Development Phase
    • Requirements Analysis - the software life cycle starts here, the goal is to specify what services the proposed system will provide, identify any conditions on those services, and define how the world will interact with the system.
    • Design - this step involves creating a plan for the construction of the proposed system.
    • Implementation - involves writing the programs, creating the data files and developing the databases.
    • Testing - testing in the past was the process of debugging programs and confirming the final product was compatible with the software requirements, however today testing may be incorporated into other steps.
    Modular Implementation separating a program's functions into independent pieces, each containing all parts needed to execute a single aspect of the functionality.
    Unified Modeling Language (UML) a collection of tools that has been developed with the object-oriented paradigm in mind.
    Use Case written descriptions of how users will perform tasks on your website
    Use Case Diagram depicts interactions between the sytem and it's actors
    Data Relationships:
    • one-to-one - where the relationship between entities is only one to one (ex. {X to Y})
    • one-to-many - where the relationship between entities is one to many (ex. {X to Y1, Y2})
    • many-to-many - where the relationship between entities is many to many (ex. {X1 to Y1, Y2, Y3} {X2 to Y1, Y2, Y3})
    Design Pattern a repeatable solution to an occuring problem in a software design
    Software Quality Assurance (SQA) groups that oversees and enforce the quality control of a system.
    Software Testing:
    • Glass Box Testing - the software tester is aware of the interior structure of the software and uses this knowledge when designing the test.
    • Black Box Testing - tests that do not rely on knowledge of the software's interior composition, testing is performed from the user's POV.
    • Beta Testing - black-box category testing, in which a preliminary version of the software is given to a segment of the intended audience to learn how to software performs in real-life.
    • Alpa Testing - similar to beta testing, except it is performed at the developer's site.
    Documentation:
    • User Documentation - its purpose is to explain the features of the software and describe how to use them.
    • System Documentation - its purpose is to describe the software's internal composition so that the software can be maintained later in its life cycle.
    • Technical Documentation - its purpose is to describe how a software system should be installed and serviced.
    Intellectual Property Law
    • Software License - a legal agreement between the owner and user of a software product that grants the user certain permissions to use the product without transferring ownership rights to the intellectual property.
    • Software Disclaimer - the developer's formal notice to the user of the software that the developer is denying any liability for damages caused from the software
    • The est defenses against software liability claims is to apply sound software engineering principles during the development of the software, use a level of care, produce and maintain records.
